角色：基于 observe 产物生成 dry-run 更新方案。
约束：只读分析和规划，不修改任何文件。

## 生成方案
请基于 replay-card.md、growth-opportunities.md、api-skill-growth-plan.md 和当前 skill 文件，生成一版 dry-run 更新方案。只分析和规划，不修改任何文件。请明确哪些内容进入 SKILL.md，哪些进入 reference，哪些进入 tool_registry，哪些进入 endpoint_manifest，哪些只进入归档或 backlog。

## 选择最合适方案
请根据现在的实际应用场景和 skill 的情况决定最合适的更新方案。不要向我提出多个选择题，先给出你认为最合适的一版 dry-run plan。

## curator 审查
请作为 curator 审查这些 growth opportunities：哪些应该进入 skill，哪些应该进入 reference，哪些应该转成 tool backlog，哪些应该归档，哪些不应该采纳。请保持克制，避免让 skill 过度膨胀。
